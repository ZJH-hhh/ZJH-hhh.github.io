项目系统设计

  • menu:菜单页面
  • playground：游戏界面
  • settings：设置界面

项目文件结构

  • templates目录：管理html文件
  • urls目录：管理路由，即链接与函数的对应关系
  • views目录：管理http函数
  • models目录：管理数据库数据
  • static目录：管理静态文件，例如：
    • css:对象的格式，例如位置，长宽，颜色，背景，字体大小等
    • js：对象的逻辑，例如对象的创建与销毁，事件函数，移动，变色等
    • images：图片
    • audio：声音
    • …
  • consumers目录：管理websocket函数

实现简易游戏引擎

首先，游戏中人物的移动是由一秒内渲染多帧来实现的，而移动的方式基本都是相同的，往某个前进方向移动多少单位距离
所以可以实现一个基类AcGameObject

  1. start() 在第一帧执行，一般是创建对象
  2. update() 在游戏的每一帧都会执行，一般是渲染当前的状态，例如位置，动作等
  3. before_destroy() 在对象被销毁前执行一次
  4. destroy() 销毁对象
